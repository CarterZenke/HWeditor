UUID Replacer and Criteria Builder

Upload a project.json to have the UUIDs replaced with new ones! If UUIDs are missing from the file, this tool will add them. It also replaces the Criteria section with a new one, ensuring that all activity UUIDs are properly in place.

Yes that's right! You no longer have to add id fields or the criteria section manually when writing Json files. And if you are using a template copied from a project that already has them, it's fine to leave them in.

Please keep in mind a few things while using this tool. The course UUID, which is the id field at the topmost level, will not be replaced if it exists. This is in case a template was linked to the course Json file when created. If the first id is not correct, just remove it and this program with create a new one.

Also, in order for this tool to correctly identify different activity/checkpoint types, the correct naming conventions must be used. Here is a list of the currently supported activities/checkpoints and how they are identified:

  • Short Answer: The block's 'variant' must be set to 'short_answer'.
  • Rubric: The block's 'variant' must be set to 'rubric'.
  • Debugging Challenge: The checkpoint's 'title' must contain 'Debugging Challenge'.
  • Project Example: The checkpoint's 'title' must contain 'Project Example'.
  • Vocabulary: The checkpoint's 'title' must contain 'Vocabulary Check'.
  • Code Review: The checkpoint's 'title' must contain 'Code Review'.
  • Core Concept Review: The checkpoint's 'title' must contain 'checkpoint-set'.
  • MCQs: The checkpoint-set's 'tool' must be 'checkpoint-set'.

If an activity does not match these criteria exactly, it will not be added to the Criteria section! I engineered these to match what I see currently being used in the json files. If something needs to be added or changed, please just let me know! It is really easy to update these.

To use, simply upload a json file.
